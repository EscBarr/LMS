1. Доступ к репозиториям

Первым шагом в процессе проверки работ студентов является просмотр, созданных ими репозиториев с исходным кодом

Репозиторий - это каталог файловой системы, в котором находятся файлы конфигурации, файлы журналов, хранящие операции, выполняемые над репозиторием, индекс, описывающий расположение файлов, и хранилище, содержащее файлы.

1.1 Устройство репозитория

  • Перейдите на вкладку "Список репозиториев студента", выбрав соответсвующий пункт в меню
  • Нажмите на название репозитория
  • В появившемся окне будет отображен список файлов проекта студента, которые можно просматривать

2. Взаимодействие с репозиторием

После перехода в репозиторий, возможны 3 основных способа взаимодействия:

  • Просмотр файлов
  • Скачивание репозитория
  • Запуск тестирования

2.1 Скачивание репозитория

  • Перейдите на вкладку "Список репозиториев студента", выбрав соответсвующий пункт в меню
  • Нажмите на название репозитория
  • В появившемся окне будет отображен список файлов проекта студента, которые можно просматривать и редактировать
  • Нажмите на кнопку "Действия"
  • Нажмите на кнопку "Скачать ZIP"
  • После этого запустится скачивание сжатого архива с файлами репозитория

2.2 Организация тестирования

Для организации тестирования необходимо наличие файл с расширением .drone.uml (подробная информация о его содержимом на сайте https://docs.drone.io/pipeline/configuration/) и выполнение следующих условий:

  • Тип сборки обязательно должен быть docker для изоляции собираемого проекта
  • Необходимо выполнения требований которые идут от системы сборки языка
  • В случае с языком C# файл drone.uml должен находиться в одной папке с .sln файлом проекта
  • В случае с языком С++ можно использовать различные системы сборки языка, необходимо настроить среду с помощью консольных команд. Так для CMake главное, чтобы файл drone.uml находился в одной папке с CMakeLists

2.3 Запуск тестирования

  • Перейдите на вкладку "Список репозиториев студента", выбрав соответсвующий пункт в меню
  • Нажмите на название репозитория
  • В появившемся окне будет отображен список файлов проекта студента, которые можно просматривать и редактировать
  • Нажмите на кнопку "Действия"
  • Нажмите на кнопку "Запуск тестирования"
  • После этого запустится тестирование проекта
  • При успешном завершении будет выведет окно с результатами тестирования, или лог ошибок
